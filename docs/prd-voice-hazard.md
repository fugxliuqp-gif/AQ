# PRD：语音报隐患 MVP

## 1. 项目背景与目标

### 1.1 背景
化工企业一线员工（操作工、巡检员）在发现现场隐患时，传统流程是：回到值班室/找到电脑 → 登录系统 → 在多层菜单中找到"隐患上报" → 填写长表单。这一过程平均耗时 8~15 分钟，导致大量隐患被口头传递而漏记。

### 1.2 目标
通过**语音/拍照优先**的移动端交互，让一线员工在 60 秒内完成隐患上报。AI 负责自动识别设备、评估风险等级、推荐整改措施，最大限度减少人工填报。

### 1.3 成功指标
| 指标 | 基线 | 目标 |
|---|---|---|
| 隐患从发现到上报耗时 | 10 分钟 | ≤ 60 秒 |
| 一线员工月度上报数/人 | 3.2 条 | ≥ 8 条 |
| 表单字段手动填写率 | 100% | ≤ 20% |
| 设备识别准确率 | - | ≥ 90% |
| 风险等级自动评估准确率 | - | ≥ 85%（人工复核后） |

## 2. 用户故事

### 2.1 主要角色
- **操作工/巡检员**：现场发现异常，需要快速上报。
- **车间安全员**：接收隐患，进行复核、分派整改。
- **系统管理员**：配置知识库、查看AI识别日志。

### 2.2 用户故事

**US-01 语音快速上报**
> 作为一名操作工，当我发现设备异常时，我想对着手机说几句话就能上报隐患，这样我可以继续作业而不必回办公室填表。

**US-02 拍照辅助识别**
> 作为一名巡检员，当我看到管道渗漏或阀门锈蚀时，我想拍一张照片并简单描述，让系统帮我定位设备并判断风险，这样我不用记住复杂的设备编号。

**US-03 AI 自动补全**
> 作为一名操作工，我希望系统能根据我的语音自动填好"设备名称、位置、风险等级、推荐措施"，我只需要点一下确认，这样可以避免我不会写字或写错字的问题。

**US-04 离线缓存**
> 作为一名巡检员，当我在罐区（信号弱）发现隐患时，我希望可以先录音/拍照，等回到有网络的地方再自动同步，这样不会因为没网而漏报。

**US-05 安全员复核**
> 作为车间安全员，我希望在系统中看到 AI 自动识别的隐患草稿，我可以快速修改风险等级和整改人后一键下达，这样我每天处理隐患的时间减少一半。

## 3. 功能范围

### 3.1 In Scope（本期做）
1. 移动端语音录入与语音转文字（STT）
2. 拍照上传与图片压缩
3. AI 意图识别：提取设备、异常现象、位置
4. AI 风险分级：基于化工行业规则（一般/重大）自动评估
5. AI 措施推荐：基于历史隐患库推荐整改措施
6. 隐患草稿确认页（用户可编辑后提交）
7. 离线缓存：语音/图片本地存储，联网后自动上传
8. 安全员 Web 端复核列表

### 3.2 Out of Scope（本期不做）
1. 视频拍摄与实时流媒体分析
2. AI 自动生成整改工单并直接推送到维修班组（本期只生成隐患单）
3. 与DCS/PLC实时数据联动判断异常（只支持人工描述）
4. 多语言语音（本期只支持中文普通话）
5. 防爆PDA硬件适配（本期只做普通安卓/iOS手机）

## 5. 功能详述

### 5.1 移动端：语音报隐患首页

**页面入口**：企业微信/钉钉/独立App 工作台中的"语音报隐患"图标。

**界面元素**：
- 顶部：当前用户姓名 + 默认车间（可切换）
- 中部：大圆形语音按钮（按住说话，松手结束）
- 底部：拍照图标 + 最近上报记录快捷入口

**交互逻辑**：
1. 长按语音按钮（≥300ms）触发录音，界面显示"正在聆听..."波纹动画
2. 松开手指停止录音，自动转文字（最长支持 60 秒语音）
3. 若识别文字为空或置信度 < 0.6，提示"未听清，请重试"
4. 识别成功后，1 秒内跳转到"草稿确认页"

**拍照逻辑**：
1. 点击拍照图标调用系统相机
2. 拍摄后支持简单裁剪（只做正方形缩放到 ≤ 2MB）
3. 图片与语音可**同时提交**（语音描述 + 照片佐证）

### 5.2 移动端：隐患草稿确认页

**页面布局**：
- 顶部：语音原文（可编辑文本框）
- 中部：AI 自动填充的信息卡片
  - 设备名称（带置信度标签：高/中/低）
  - 所在位置（车间/工段/位号）
  - 异常描述（AI 结构化摘要）
  - 风险等级（一般隐患 / 重大隐患，带颜色标签）
  - 推荐整改措施（可下拉换其他历史措施）
- 底部："确认上报" 主按钮 + "重新描述" 次按钮

**用户操作**：
- 点击任一字段可手动修正
- 若设备识别置信度为"低"，该字段标红并提示"请核对"
- 支持补充拍照（最多 3 张）

### 5.3 AI 识别逻辑

#### 5.3.1 设备识别
**输入**：语音转文字结果 + 用户当前车间
**输出**：设备编号、设备名称、位号、置信度

**识别策略**：
1. **精确匹配**：语音中出现完整设备编号（如"P-203"、"R101"）→ 直接命中
2. **别名匹配**：语音中出现设备别名（如"三号泵"、"加氢釜"）→ 通过同义词库匹配
3. **位置+描述推断**：如"三车间那个循环泵" → 结合用户权限范围内的设备列表推断
4. **置信度规则**：
   - 精确匹配 ≥ 0.95
   - 别名匹配 ≥ 0.85
   - 推断匹配 0.70~0.84（标黄提示人工核对）
   - < 0.70 标红，要求用户手动选择

#### 5.3.2 风险等级自动评估
**输入**：设备类型、异常现象描述、关键词
**输出**：一般隐患 / 重大隐患 + 依据条款

**评估规则（基于化工行业通用标准）**：
| 触发关键词/条件 | 风险等级 |
|---|---|
| 泄漏、渗漏、滴漏 + 危化品介质 | 重大隐患 |
| 安全阀失效、爆破片超期、压力表损坏 | 重大隐患 |
| 温度/压力异常波动 + 涉及反应釜 | 重大隐患 |
| 普通跑冒滴漏（非危化品） | 一般隐患 |
| 设备异响、振动、温度偏高（未超限） | 一般隐患 |
| 防护栏缺失、标识模糊、地面油污 | 一般隐患 |

**AI 增强**：
- 使用 RAG 检索企业历史隐患库中相似案例的定级结果
- 若历史案例中 80% 以上为"重大"，则本次也推荐"重大"并提示"历史上同类异常多定为重大隐患"

#### 5.3.3 整改措施推荐
**策略**：
1. 优先匹配该设备近 6 个月内同类异常的实际整改措施
2. 次选该企业同类型设备的通用整改措施库
3. 最后 fallback 到化工行业标准措施模板

### 5.4 离线缓存机制

**触发条件**：
- 点击"确认上报"时检测到网络不可用（navigator.onLine === false 或 API 超时）

**处理逻辑**：
1. 语音文件（MP3/AMR）、压缩后的图片、识别文本存入 IndexedDB（或 SQLite）
2. 本地队列管理，显示"已缓存，待联网后自动上传"
3. 监听网络恢复事件，自动批量上传（最多同时上传 2 个文件，避免拥塞）
4. 上传成功后本地删除缓存，并推送本地通知

### 5.5 Web端：安全员复核台

**列表页**：
- 筛选条件：今日待复核 / 已复核 / AI推荐重大隐患 / 低置信度
- 列表字段：上报时间、上报人、设备、AI风险等级、复核状态
- 快捷操作：一键通过 / 打开详情

**详情页**：
- 左侧：语音播放（支持倍速）、图片放大查看、AI识别结果
- 右侧：可编辑表单（隐患级别、整改责任人、整改期限、整改措施）
- 底部："确认并下达整改" 按钮

**低置信度强提示**：
- 若设备识别置信度 < 0.85，详情页顶部显示黄色横幅："AI 对设备识别不确定，请重点核对"

## 6. 非功能需求

### 6.1 性能
- 语音转文字：≤ 3 秒（60秒内语音）
- AI 识别与填充：≤ 2 秒（不含网络延迟）
- 页面首屏加载：≤ 1.5 秒（3G网络下）
- 图片压缩：≤ 1 秒（单张 4MB → 2MB）

### 6.2 安全与合规
- 语音与图片传输使用 HTTPS
- 敏感区域照片（如涉及工艺参数）需加水印并限制转发
- AI 推荐的风险等级和措施必须**可溯源**，显示引用来源（历史隐患单号或标准条款）
- 所有 AI 识别结果需人工确认后才能成为正式隐患记录（满足审计要求）

### 6.3 兼容性
- 移动端：Android 8+ / iOS 13+
- 支持主流浏览器：Chrome、Edge、Safari、企业微信内置浏览器

### 6.4 可靠性
- 语音录制中断（如来电）自动保存已录部分并提示
- 上传失败重试 3 次，仍失败则保留本地缓存

## 8. 验收标准

### 8.1 功能验收
- [ ] 操作工长按语音按钮 3 秒说话，松手后 3 秒内看到文字
- [ ] 说出"三号泵轴承有异响"，AI 正确识别设备为 P-203（置信度 ≥ 0.85）
- [ ] 说出"甲醇储罐底部有渗漏"，AI 正确评估为"重大隐患"
- [ ] 在无网络环境下录音，回到有网络环境后 30 秒内自动同步成功
- [ ] 安全员在 Web 端打开隐患详情，能看到语音播放、图片、AI识别来源

### 8.2 性能验收
- [ ] 语音转文字平均耗时 ≤ 3 秒（测试样本 100 条）
- [ ] AI 识别接口平均耗时 ≤ 2 秒（测试样本 100 条）
- [ ] 移动端首屏加载时间 ≤ 1.5 秒

### 8.3 用户体验验收
- [ ] 5 名一线操作工在无人指导情况下，能独立完成一次语音上报
- [ ] 80% 测试用户认为"比原来的表单填报快得多"

## 9. 风险与应对

| 风险 | 影响 | 应对措施 |
|---|---|---|
| 现场噪音大，语音识别准确率低 | 高 | 支持近场麦克风降噪；允许用户修改文字；关键设备用手选兜底 |
| 化工术语/设备别名多，AI识别不准 | 高 | 建设企业级设备同义词库；持续收集 badcase 微调模型 |
| 员工对AI不信任，拒绝使用 | 中 | 所有AI结果需人工确认；强调"AI辅助，人做决策" |
| 防爆区禁止普通手机使用 | 中 | MVP阶段只在非防爆区推广；二期适配防爆PDA |
| 离线缓存数据过多导致手机存储满 | 低 | 设置最大缓存阈值（如 100MB），超限提醒清理 |
